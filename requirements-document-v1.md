## 一、项目概述

### 1.1 项目名称
**热点电商自动化流水线** (Hot E-commerce Automation Pipeline)

### 1.2 项目目标
自动抓取社交平台热点 → AI视频创作 → 产品带货 → 智能客服 → 自动发货

### 1.3 项目愿景
打造一个高效的社交媒体内容自动创作与电商变现系统，通过AI技术实现从热点发现到产品销售的全流程自动化。

### 1.4 MVP 定义
**最小可行产品 (MVP)**：
```
微博热点监控 → AI视频生成(模板化) → 手动产品关联 → 发布到测试账号
```

## 二、系统架构

### 2.1 整体架构
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  热点抓取   │ ──> │  AI视频创作  │ ──> │   电商带货   │
│  模块       │     │    模块      │     │    模块      │
└─────────────┘     └─────────────┘     └─────────────┘
      │                   │                   │
      v                   v                   v
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  数据库层   │     │   内容审核   │     │   智能客服   │
│  PostgreSQL │     │    模块      │     │    模块      │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 2.2 技术栈
| 模块 | 技术方案 |
|------|----------|
| 后端框架 | Python (FastAPI + Scrapy + Celery) |
| 数据库 | PostgreSQL (主) + Redis (缓存/队列) |
| 部署 | Docker + Docker Compose |
| AI视频 | 模板化(FFmpeg) → AI生成(RunwayML/Stable Video) |
| 客服系统 | Rasa框架 + 规则引擎 |

## 三、功能模块详细需求

### 3.1 热点抓取模块

#### 3.1.1 功能概述
自动监控微博、小红书、Bilibili 的热点内容，为AI视频创作提供素材。

#### 3.1.2 数据源优先级
| 平台 | 优先级 | 抓取方式 | 说明 |
|------|--------|----------|------|
| 微博 | ⭐⭐⭐ 最高 | API + 爬虫 | API相对开放 |
| Bilibili | ⭐⭐ 次优先 | API + 爬虫 | 年轻用户群体 |
| 小红书 | ⭐ 最后 | 爬虫(需额外策略) | 反爬虫严格 |

#### 3.1.3 功能需求

**F-HOT-001: 热点数据抓取**
- 系统应每小时自动抓取一次各平台热点
- 系统应支持微博热搜榜 API 获取
- 系统应支持B站热门视频 API 获取
- 系统应记录抓取时间、热点内容、热度指数

**F-HOT-002: 数据去重机制**
- 系统应基于热点ID的哈希值判断重复
- 系统应跳过已处理的热点
- 系统应记录处理状态

**F-HOT-003: 热点数据存储**
- 热点数据应存储到 PostgreSQL 数据库
- 数据字段：id, platform, title, url, heat_score, created_at, status

**F-HOT-004: 频率控制**
- 系统每日生成视频不超过10条
- 系统应实现防过度抓取机制

---

### 3.2 AI视频创作模块

#### 3.2.1 功能概述
基于热点内容自动生成营销视频，实现内容创作的自动化。

#### 3.2.2 MVP 功能需求

**F-VIDEO-001: 热点转视频工作流**
- 系统应支持热点内容 → 视频模板的转换
- 系统应支持文字、图片、音视频素材的合成
- 系统应输出 MP4 格式视频

**F-VIDEO-002: 模板化视频生成 (MVP阶段)**
- 系统应提供预设视频模板库
- 系统应支持模板参数配置（文字、颜色、配图）
- 系统应使用 FFmpeg 进行视频合成

**F-VIDEO-003: AI视频生成 (第二阶段)**
- 系统应集成 RunwayML 或 Stable Video API
- 系统应支持更高质量的AI生成效果

**F-VIDEO-004: 合规性标识**
- 所有生成视频必须嵌入 "AI生成" 标识
- 标识位置：视频角落 + 片尾字幕
- 标识样式：白字黑边，保证可读性

**F-VIDEO-005: 内容审核**
- 视频生成前：AI预审热点内容
- 视频生成后：人工审核确认
- 发布前：最终合规检查

---

### 3.3 电商带货模块

#### 3.3.1 功能概述
实现产品上架、促销推广、交易处理的自动化。

#### 3.3.2 电商平台优先级
| 平台 | 优先级 | 说明 |
|------|--------|------|
| 淘宝 | ⭐⭐⭐ 第一 | API成熟，生态完善 |
| 微信小店 | ⭐⭐ 第二 | 社交电商优势 |
| 京东 | ⭐ 第三 | 正规化要求高 |

#### 3.3.3 功能需求

**F-ECOM-001: 产品管理**
- 系统应支持产品信息录入
- 系统应支持产品与热点的关联
- 系统应支持产品列表展示

**F-ECOM-002: 热点-产品关联 (MVP: 手动)**
- 运营人员手动选择热点关联的产品
- 系统记录关联关系
- 未来版本：自动关联推荐

**F-ECOM-003: 交易处理**
- 系统应支持订单创建、查询、管理
- 系统应集成支付接口
- 系统应实现订单状态跟踪

---

### 3.4 智能客服模块

#### 3.4.1 功能概述
提供24/7自动客服服务，处理用户咨询和订单查询。

#### 3.4.2 技术方案
- **框架**: Rasa 框架 + 规则引擎
- **交互方式**: 文本对话
- **部署**: 独立微服务

#### 3.4.3 功能需求

**F-CS-001: 常见问题回复**
- 系统应能回复订单查询类问题
- 系统应能回复产品咨询类问题
- 系统应能回复退换货政策类问题
- 系统应提供多轮对话能力

**F-CS-002: 订单查询**
- 用户输入订单号可查询订单状态
- 用户输入手机号可查询历史订单
- 系统应展示物流信息

**F-CS-003: 自动回复模板**
- 系统应提供客服话术库
- 话术库应支持分类：售前、售中、售后
- 话术应支持变量替换（订单号、产品名等）

**F-CS-004: 转人工机制**
- 系统无法回答时应转人工处理
- 系统应记录转人工会话

---

### 3.5 物流自动化模块

#### 3.5.1 功能概述
实现自动打单、发货、物流跟踪的全流程自动化。

#### 3.5.2 功能需求

**F-LOG-001: 订单推送**
- 新订单自动推送到物流系统
- 支持批量处理

**F-LOG-002: 电子面单**
- 系统应自动生成电子面单
- 支持主流快递：顺丰、圆通、中通、韵达

**F-LOG-003: 物流跟踪**
- 系统应自动跟踪物流状态
- 状态更新推送给用户

## 四、触发机制

### 4.1 半自动模式 (MVP)
```
热点抓取(自动) → 人工确认(人工) → 视频生成(自动) → 人工审核(人工) → 发布(自动)
```

| 操作 | 频率 | 限制 |
|------|------|------|
| 热点抓取 | 每小时1次 | 无 |
| 视频生成 | 每日不超过10条 | 系统自动控制 |

## 五、数据模型

### 5.1 热点表 (hot_topics)
```sql
CREATE TABLE hot_topics (
    id SERIAL PRIMARY KEY,
    platform VARCHAR(50) NOT NULL,      -- 平台: weibo, bilibili, xiaohongshu
    topic_id VARCHAR(100) UNIQUE NOT NULL, -- 平台原始ID
    title TEXT NOT NULL,                -- 热点标题
    url TEXT,                           -- 原始链接
    heat_score INTEGER DEFAULT 0,       -- 热度指数
    hash_value VARCHAR(64),             -- 去重哈希
    status VARCHAR(20) DEFAULT 'pending', -- pending, processing, completed, skipped
    created_at TIMESTAMP DEFAULT NOW(),
    processed_at TIMESTAMP
);
```

### 5.2 视频表 (videos)
```sql
CREATE TABLE videos (
    id SERIAL PRIMARY KEY,
    topic_id INTEGER REFERENCES hot_topics(id),
    file_path TEXT,
    status VARCHAR(20) DEFAULT 'draft', -- draft, reviewing, approved, published
    ai_labeled BOOLEAN DEFAULT TRUE,    -- 是否有AI标识
    created_at TIMESTAMP DEFAULT NOW(),
    published_at TIMESTAMP
);
```

### 5.3 产品表 (products)
```sql
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    price DECIMAL(10, 2),
    platform VARCHAR(50),              -- 淘宝/京东/微信
    external_id VARCHAR(100),          -- 外部平台产品ID
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 5.4 订单表 (orders)
```sql
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    order_no VARCHAR(50) UNIQUE NOT NULL,
    product_id INTEGER REFERENCES products(id),
    customer_name VARCHAR(100),
    phone VARCHAR(20),
    address TEXT,
    status VARCHAR(20) DEFAULT 'pending', -- pending, paid, shipped, delivered
    created_at TIMESTAMP DEFAULT NOW(),
    shipped_at TIMESTAMP
);
```

## 六、合规要求

### 6.1 数据安全
- 所有用户数据存储在**境内服务器**
- 数据传输使用 HTTPS 加密
- 敏感信息(手机号、地址)加密存储

### 6.2 AI内容披露
- 所有AI生成视频必须嵌入 "**AI生成**" 标识
- 标识位置：视频右下角 + 片尾字幕
- 字体：白色黑边，清晰可读

### 6.3 内容审核
**三级审核机制**：
1. **AI预审**：基于关键词和图片识别过滤违规内容
2. **人工审核**：运营人员确认热点和视频内容
3. **发布前检查**：最终合规性确认

### 6.4 平台规则
- 遵守各平台 API 使用条款
- 遵守社交媒体内容发布规范
- 定期更新合规检查清单
