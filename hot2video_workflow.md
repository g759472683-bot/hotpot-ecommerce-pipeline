## 二、各阶段详细说明

### 阶段1：热点抓取
- **频率**：每小时一次（Cron Job）
- **防重复**：热点ID做MD5哈希，存入数据库前查重
- **关联度评分**：基于关键词匹配（产品词、品类词）

### 阶段2：人工确认
- **界面**：Django/Flask后台，热点列表页
- **操作**：勾选热点 → 选择关联产品 → 点击"生成视频"
- **阈值**：每日最多生成10条（预算控制）

### 阶段3：Prompt构造
```
模板：
"镜头[运动方式]，[主体]在[场景]中[动作]，[氛围描述]，[色彩风格]"

示例（微博热点："某明星代言奶茶"）：
"镜头从明星手持奶茶的特写缓缓拉远，展示奶茶杯上的品牌logo，氛围时尚清新，色调明亮温暖。"
```

### 阶段4：视频生成
- **API调用**：MiniMax Hailuo（主），快手可灵（备）
- **超时处理**：60秒无响应自动切换备选
- **重试机制**：最多重试3次，失败后标记并告警

### 阶段5：后处理（FFmpeg）
```bash
# 1. 嵌入"AI生成"水印标识
drawtext=text='内容由AI生成':fontsize=36:fontcolor=white@0.8:x=(w-text_w)/2:y=h-50

# 2. 添加字幕（热点标题）
drawtext=text='热点：XXX':fontsize=24:fontcolor=yellow:x=20:y=20

# 3. 拼接背景音乐（可选）
ffmpeg -i video.mp4 -i bgm.mp3 -c:v copy -c:a aac -shortest output_final.mp4
```

### 阶段6：发布
- **微博**：开放平台API（需申请权限）
- **B站**：创作中心API（需申请）
- **发布后记录**：视频URL、发布时间、平台反馈

## 三、每日上限控制

```
每日视频生成上限：10条
每小时触发上限：2条
单条视频重试次数：3次
超时等待时间：60秒
```

---

## 四、异常处理

| 异常场景 | 处理方式 |
|---|---|
| API调用失败 | 自动切换备选API，失败则跳过 |
| 视频生成超时 | 60秒超时，自动重试3次 |
| 热点重复 | MD5哈希查重，已处理过则跳过 |
| 超过日限额 | 拒绝生成，加入次日队列 |
| 内容不合规 | 标记告警，人工复核 |
